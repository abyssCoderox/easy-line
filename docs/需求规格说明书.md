## 1. 引言

### 1.1 编写目的

本文档描述LINE Bot Demo版本的功能需求，目标是在一周内完成一个可演示的聊天交互系统，包含定时消息推送功能。

### 1.2 项目背景

本项目旨在快速构建一个LINE Bot Demo，验证核心技术可行性，展示LINE平台与大模型集成的聊天交互能力，以及定时消息推送功能。

### 1.3 术语定义

| 术语 | 定义 |
|------|------|
| LINE Bot | 基于LINE Messaging API开发的聊天机器人 |
| @line/bot-sdk | LINE官方提供的Node.js SDK |
| Webhook | LINE服务器向开发者服务器推送消息的HTTP回调机制 |
| LangChain | 大模型应用开发框架 |
| node-cron | Node.js定时任务调度库 |

---

## 2. 系统概述

### 2.1 系统目标

在**一周内**完成一个可演示的LINE Bot Demo，实现：
- LINE平台消息正常收发
- 大模型智能对话功能
- 定时消息推送功能
- 完整的聊天交互演示流程

### 2.2 开发原则

| 原则 | 说明 |
|------|------|
| 🎯 核心优先 | 聚焦核心功能，确保聊天交互和定时推送跑通 |
| 🚀 快速迭代 | 每日验证，快速调整 |
| 📦 精简架构 | 最小化中间件，只保留必要组件 |
| 🔧 成熟框架 | 使用@line/bot-sdk、LangChain等成熟框架加速开发 |
| ⚡ 暂缓优化 | 安全防护、性能优化暂不作为重点 |

### 2.3 运行环境

| 环境要素 | 要求 |
|---------|------|
| 运行时 | Node.js 18+ |
| 网络 | 公网可访问（支持HTTPS或ngrok穿透） |
| 外部服务 | LINE Messaging API、OpenAI API |

## 3. 功能需求

### 3.1 LINE消息交互模块

#### 3.1.1 功能描述

使用 `@line/bot-sdk` 实现与LINE平台的消息收发交互，确保用户发送的消息能够被正确接收并回复。

#### 3.1.2 功能清单

| 功能编号 | 功能名称 | 功能描述 | 优先级 | SDK方法 |
|---------|---------|---------|--------|---------|
| F1.1 | Webhook消息接收 | 接收LINE服务器推送的消息 | 高 | `middleware()` |
| F1.2 | 签名验证 | 验证请求来自LINE服务器 | 高 | `middleware()` 自动处理 |
| F1.3 | 消息解析 | 解析文本消息内容 | 高 | `event.message.text` |
| F1.4 | 消息回复 | 通过Reply API回复用户消息 | 高 | `client.replyMessage()` |
| F1.5 | 基础错误处理 | 异常情况下不崩溃 | 中 | try-catch |

#### 3.1.3 业务流程

```
┌─────────────────────────────────────────────────────────────┐
│                    消息交互流程                              │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  用户发送消息                                                │
│       │                                                     │
│       ▼                                                     │
│  LINE服务器                                                 │
│       │                                                     │
│       │ Webhook POST                                        │
│       ▼                                                     │
│  ┌─────────────┐                                            │
│  │ middleware  │ ← SDK自动验证签名                          │
│  │ 签名验证    │                                            │
│  └──────┬──────┘                                            │
│         │                                                   │
│         ▼                                                   │
│  ┌─────────────┐                                            │
│  │ 解析消息    │                                            │
│  │ event.body │                                            │
│  └──────┬──────┘                                            │
│         │                                                   │
│         ▼                                                   │
│  ┌─────────────┐                                            │
│  │ 调用LLM     │                                            │
│  │ 生成回复    │                                            │
│  └──────┬──────┘                                            │
│         │                                                   │
│         ▼                                                   │
│  ┌─────────────┐                                            │
│  │ replyMessage│                                            │
│  │ 发送回复    │                                            │
│  └──────┬──────┘                                            │
│         │                                                   │
│         ▼                                                   │
│  用户收到回复                                                │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 3.2 大模型对话模块

#### 3.2.1 功能描述

集成LangChain框架，实现与大模型的对话交互，支持上下文记忆。

#### 3.2.2 功能清单

| 功能编号 | 功能名称 | 功能描述 | 优先级 |
|---------|---------|---------|--------|
| F2.1 | LangChain集成 | 集成LangChain框架 | 高 |
| F2.2 | 对话生成 | 调用大模型生成回复 | 高 |
| F2.3 | 上下文管理 | 记忆最近N条对话历史 | 中 |
| F2.4 | 降级回复 | API异常时返回预设回复 | 中 |

#### 3.2.3 上下文管理规则

- 默认保留最近 **3条** 对话记录作为上下文
- 使用内存存储（Map结构）
- 按用户ID分组管理

---

### 3.3 定时消息推送模块

#### 3.3.1 功能描述

使用 `node-cron` 实现定时任务调度，配合 `@line/bot-sdk` 的 `pushMessage` / `multicast` API 实现主动消息推送。

#### 3.3.2 功能清单

| 功能编号 | 功能名称 | 功能描述 | 优先级 | 实现方式 |
|---------|---------|---------|--------|---------|
| F3.1 | 定时任务配置 | JSON文件配置定时任务 | 高 | `config/tasks.json` |
| F3.2 | Cron调度 | 按Cron表达式执行任务 | 高 | `node-cron` |
| F3.3 | API数据获取 | 调用第三方API获取数据 | 中 | `axios` |
| F3.4 | 消息模板渲染 | 将数据填充到消息模板 | 中 | 字符串替换 |
| F3.5 | 消息推送 | 向指定用户推送消息 | 高 | `client.pushMessage()` |
| F3.6 | 批量推送 | 向多用户同时推送 | 中 | `client.multicast()` |

#### 3.3.3 业务流程

```
┌─────────────────────────────────────────────────────────────┐
│                    定时推送流程                              │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  服务启动                                                    │
│       │                                                     │
│       ▼                                                     │
│  ┌─────────────┐                                            │
│  │ 加载任务配置 │ ← config/tasks.json                       │
│  └──────┬──────┘                                            │
│         │                                                   │
│         ▼                                                   │
│  ┌─────────────┐                                            │
│  │ 注册Cron任务 │ ← node-cron                               │
│  └──────┬──────┘                                            │
│         │                                                   │
│         │ 到达执行时间                                       │
│         ▼                                                   │
│  ┌─────────────┐                                            │
│  │ 调用第三方API│ ← 获取数据                                 │
│  └──────┬──────┘                                            │
│         │                                                   │
│         ▼                                                   │
│  ┌─────────────┐                                            │
│  │ 模板渲染    │ ← 填充数据到消息模板                        │
│  └──────┬──────┘                                            │
│         │                                                   │
│         ▼                                                   │
│  ┌─────────────┐                                            │
│  │ pushMessage │ ← @line/bot-sdk                            │
│  │ 或 multicast│                                            │
│  └──────┬──────┘                                            │
│         │                                                   │
│         ▼                                                   │
│  用户收到定时推送消息                                         │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

#### 3.3.4 任务配置示例

```json
{
  "tasks": [
    {
      "id": "weather-push",
      "name": "每日天气推送",
      "enabled": true,
      "schedule": "0 8 * * *",
      "api": {
        "url": "https://api.weather.com/v1/current",
        "method": "GET",
        "headers": {
          "Authorization": "Bearer ${WEATHER_API_KEY}"
        }
      },
      "template": "今日天气: {weather}, 温度: {temp}°C",
      "targets": ["U1234567890"]
    }
  ]
}
```

## 9. 附录

```bash
# .env
PORT=3000

# LINE配置
LINE_CHANNEL_SECRET=your-channel-secret
LINE_CHANNEL_ACCESS_TOKEN=your-access-token

# OpenAI配置
OPENAI_API_KEY=your-openai-api-key

# 第三方API（可选）
WEATHER_API_KEY=your-weather-api-key
```

### 9.2 LINE Developers 配置

1. 创建 LINE Developers 账号
2. 创建 Provider 和 Channel
3. 获取 Channel Secret 和 Channel Access Token
4. 配置 Webhook URL

### 9.3 ngrok 配置（本地测试）

```bash
# 安装ngrok
npm install -g ngrok

# 启动隧道
ngrok http 3000

# 将生成的URL配置到LINE Webhook
# 例如: https://xxx.ngrok.io/webhook
```
