- ✅ 完整的聊天交互演示流程

| 原则 | 说明 |
|------|------|
| 🎯 核心优先 | 聚焦核心功能，确保聊天交互跑通 |

| 🔧 成熟框架 | 使用LangChain等成熟框架加速开发 |
| ⚡ 暂缓优化 | 安全防护、性能优化暂不作为重点 |

| 包含内容 | 暂不包含 |


**Webhook处理:**
```typescript
import { WebhookEvent, WebhookRequestBody } from '@line/bot-sdk';
import { lineService } from '../services/line';
import { llmService } from '../services/llm';

export async function webhookHandler(req: any, res: any) {
  const body: WebhookRequestBody = req.body;
  
  try {
    const events = body.events;
    const results = await Promise.all(
      events.map(event => handleEvent(event))
    );
    res.json({ status: 'ok', results });
  } catch (error) {
    console.error('Webhook error:', error);
    res.status(500).json({ status: 'error' });
  }
}

async function handleEvent(event: WebhookEvent) {
  if (event.type !== 'message' || event.message.type !== 'text') {
    return null;
  }
  
  const userMessage = event.message.text;
  const replyToken = event.replyToken;
  
  const reply = await llmService.chat(userMessage);
  await lineService.replyMessage(replyToken, reply);
  
  return { message: userMessage, reply };
}
```

#### LangChain集成代码

```typescript
import { ChatOpenAI } from '@langchain/openai';
import { ConversationChain } from 'langchain/chains';
import { BufferMemory } from 'langchain/memory';
import { config } from '../config';

export class LLMService {
  private chain: ConversationChain;
  private sessions: Map<string, BufferMemory> = new Map();
  
  constructor() {
    this.chain = new ConversationChain({
      llm: new ChatOpenAI({
        openAIApiKey: config.openai.apiKey,
        modelName: 'gpt-3.5-turbo',
        temperature: 0.7,
      }),
      memory: new BufferMemory({
        returnMessages: true,
        memoryKey: 'chat_history',
      }),
    });
  }
  
  async chat(userId: string, message: string): Promise<string> {
    try {
      const response = await this.chain.call({
        input: message,
      });
      return response.response;
    } catch (error) {
      console.error('LLM error:', error);
      return '抱歉，我暂时无法回答，请稍后再试。';
    }
  }
}

export const llmService = new LLMService();
```

## 8. 风险与应对

### 8.1 主要风险

| 风险 | 可能性 | 应对措施 |
|------|--------|---------|
| LINE API配置问题 | 中 | 提前验证配置，准备备用方案 |
| OpenAI API限制 | 低 | 准备降级回复，控制调用频率 |
| 内网穿透不稳定 | 中 | 准备多个ngrok账号或使用云服务器 |
| 时间不足 | 中 | 优先核心功能，非必要功能砍掉 |

### 8.2 降级方案

| 场景 | 降级方案 |
|------|---------|
| 大模型API不可用 | 返回预设回复 |
| Webhook验证失败 | 返回简单确认消息 |
| 服务异常 | 自动重启，记录日志 |
