## 1. 项目背景

### 1.1 需求背景

当前 LINE Bot 系统已实现基础的聊天对话和定时任务功能，但存在以下问题：
- 意图识别与工具调用分离，代码耦合度高
- 新增工具需要手动编写匹配逻辑
- 未充分利用 LangChain 框架能力

### 1.2 项目目标

基于 LangChain Agent 重构消息处理流程，实现：
- 统一的 Agent 入口处理所有用户消息
- 工具（Tool）标准化封装，支持动态扩展
- 任务管理功能封装为 Tool，由 Agent 自动路由
- 第三方工具调用（告警查询等）由 Agent 自动决策

### 1.3 适用范围

本设计适用于 LINE Bot 系统的全面重构，影响：
- `/api/chat` REST API 接口
- LINE Webhook 消息处理
- 任务管理服务（封装为 Tool）

### 3.2 模块设计

#### 3.2.1 AgentService（核心服务）

**职责**：统一消息处理入口，管理 Agent 执行

**核心方法**：
| 方法 | 描述 |
|------|------|
| `run(input, userId)` | 执行 Agent 处理用户消息 |
| `getHistory(userId)` | 获取用户会话历史 |
| `clearHistory(userId)` | 清除用户会话历史 |

#### 3.2.2 Tools（工具集）

| 工具名称 | 描述 | 调用服务 |
|---------|------|---------|
| `create_task` | 创建定时任务 | TaskManagerService |
| `list_tasks` | 查看任务列表 | TaskManagerService |
| `delete_task` | 删除任务 | TaskManagerService |
| `enable_task` | 启用任务 | TaskManagerService |
| `disable_task` | 禁用任务 | TaskManagerService |
| `alert_query` | 查询告警信息 | 第三方 API |
| `device_status` | 查询设备状态 | 第三方 API |

### 3.3 数据流

```
1. 用户输入: "每天早上9点提醒我开会"
        │
        ▼
2. AgentService.run()
        │
        ▼
3. Agent 分析 ──► 决策：调用 create_task 工具
        │
        ▼
4. create_task Tool 执行

        ▼

        │
        ▼
6. Agent 生成回复 ──► "已为您创建定时任务！"

        ▼
```

## 8. 错误处理

### 8.1 错误场景

| 场景 | 处理方式 |
|------|---------|
| Agent 执行超时 | 返回 "处理超时，请稍后重试" |
| 工具执行失败 | Agent 生成友好的错误提示 |
| LLM API 错误 | 返回降级回复 |
| 无匹配工具 | Agent 直接对话回复 |

### 8.2 降级策略

```typescript
async run(input: string, userId: string): Promise<string> {
  try {
    return await this.agentExecutor.invoke({ input });
  } catch (error) {
    // 降级：使用基础 LLM 对话
    return await this.fallbackChat(input, userId);
  }
}
```
