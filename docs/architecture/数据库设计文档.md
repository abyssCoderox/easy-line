## 1. 数据库概述

### 1.1 数据库选型

| 项目 | 选型 |
|------|------|
| 数据库类型 | PostgreSQL 14+ |
| 字符集 | UTF-8 |
| 排序规则 | en_US.UTF-8 |

### 1.2 命名规范

| 对象类型 | 命名规范 | 示例 |
|---------|---------|------|
| 表名 | 小写下划线分隔，复数形式 | `users`, `task_configs` |
| 字段名 | 小写下划线分隔 | `user_id`, `created_at` |
| 主键 | `id` | `id` |
| 外键 | `{表名单数}_id` | `user_id`, `task_id` |
| 索引 | `idx_{表名}_{字段名}` | `idx_users_line_user_id` |
| 唯一索引 | `uq_{表名}_{字段名}` | `uq_users_line_user_id` |

## 3. 数据表设计

### 3.1 用户表 (users)

存储LINE用户基本信息。

| 字段名 | 数据类型 | 约束 | 默认值 | 说明 |
|--------|---------|------|--------|------|
| id | UUID | PK, NOT NULL | gen_random_uuid() | 主键 |
| line_user_id | VARCHAR(64) | NOT NULL, UNIQUE | - | LINE用户ID |
| display_name | VARCHAR(255) | - | - | 用户昵称 |
| picture_url | VARCHAR(512) | - | - | 头像URL |
| status | VARCHAR(20) | NOT NULL | 'active' | 状态: active, blocked |
| created_at | TIMESTAMP WITH TIME ZONE | NOT NULL | NOW() | 创建时间 |
| updated_at | TIMESTAMP WITH TIME ZONE | NOT NULL | NOW() | 更新时间 |

**索引：**
- `uq_users_line_user_id` - line_user_id唯一索引
- `idx_users_status` - status索引

**SQL定义：**
```sql
CREATE TABLE users (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    line_user_id VARCHAR(64) NOT NULL,
    display_name VARCHAR(255),
    picture_url VARCHAR(512),
    status VARCHAR(20) NOT NULL DEFAULT 'active',
    created_at TIMESTAMP WITH TIME ZONE NOT NULL DEFAULT NOW(),
    updated_at TIMESTAMP WITH TIME ZONE NOT NULL DEFAULT NOW(),
    CONSTRAINT uq_users_line_user_id UNIQUE (line_user_id)
);

CREATE INDEX idx_users_status ON users(status);
```

### 3.2 对话记录表 (conversations)

存储用户与Bot的对话历史。

| 字段名 | 数据类型 | 约束 | 默认值 | 说明 |
|--------|---------|------|--------|------|
| id | UUID | PK, NOT NULL | gen_random_uuid() | 主键 |
| user_id | UUID | FK, NOT NULL | - | 用户ID |
| role | VARCHAR(20) | NOT NULL | - | 角色: user, assistant |
| content | TEXT | NOT NULL | - | 消息内容 |
| intent_id | VARCHAR(50) | - | - | 识别的意图ID |
| created_at | TIMESTAMP WITH TIME ZONE | NOT NULL | NOW() | 创建时间 |

**索引：**
- `idx_conversations_user_id` - user_id索引
- `idx_conversations_created_at` - created_at索引

**SQL定义：**
```sql
CREATE TABLE conversations (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    role VARCHAR(20) NOT NULL,
    content TEXT NOT NULL,
    intent_id VARCHAR(50),
    created_at TIMESTAMP WITH TIME ZONE NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_conversations_user_id ON conversations(user_id);
CREATE INDEX idx_conversations_created_at ON conversations(created_at);
```

### 3.3 意图配置表 (intents)

存储意图识别规则配置。

| 字段名 | 数据类型 | 约束 | 默认值 | 说明 |
|--------|---------|------|--------|------|
| id | UUID | PK, NOT NULL | gen_random_uuid() | 主键 |
| intent_id | VARCHAR(50) | NOT NULL, UNIQUE | - | 意图标识 |
| intent_name | VARCHAR(100) | NOT NULL | - | 意图名称 |
| keywords | JSONB | - | '[]' | 关键词列表 |
| patterns | JSONB | - | '[]' | 正则表达式列表 |
| handler | VARCHAR(100) | NOT NULL | - | 处理器类名 |
| priority | INTEGER | NOT NULL | 0 | 优先级 |
| enabled | BOOLEAN | NOT NULL | TRUE | 是否启用 |
| created_at | TIMESTAMP WITH TIME ZONE | NOT NULL | NOW() | 创建时间 |
| updated_at | TIMESTAMP WITH TIME ZONE | NOT NULL | NOW() | 更新时间 |

**索引：**
- `uq_intents_intent_id` - intent_id唯一索引
- `idx_intents_enabled_priority` - enabled和priority复合索引

**SQL定义：**
```sql
CREATE TABLE intents (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    intent_id VARCHAR(50) NOT NULL,
    intent_name VARCHAR(100) NOT NULL,
    keywords JSONB DEFAULT '[]',
    patterns JSONB DEFAULT '[]',
    handler VARCHAR(100) NOT NULL,
    priority INTEGER NOT NULL DEFAULT 0,
    enabled BOOLEAN NOT NULL DEFAULT TRUE,
    created_at TIMESTAMP WITH TIME ZONE NOT NULL DEFAULT NOW(),
    updated_at TIMESTAMP WITH TIME ZONE NOT NULL DEFAULT NOW(),
    CONSTRAINT uq_intents_intent_id UNIQUE (intent_id)
);

CREATE INDEX idx_intents_enabled_priority ON intents(enabled, priority DESC);
```

## 4. Redis数据结构设计

### 4.1 对话上下文缓存

用于存储用户最近的对话上下文，提高大模型调用效率。

| Key模式 | 数据类型 | 过期时间 | 说明 |
|---------|---------|---------|------|
| `context:{user_id}` | List | 30分钟 | 用户对话上下文队列 |

**数据结构：**
```
Key: context:U1234567890abcdef
Value: [
  {"role": "user", "content": "你好", "timestamp": 1700000000},
  {"role": "assistant", "content": "你好！有什么可以帮助你的吗？", "timestamp": 1700000001}
]
TTL: 1800 (30分钟)
```

### 4.2 意图配置缓存

缓存意图配置，减少数据库查询。

| Key模式 | 数据类型 | 过期时间 | 说明 |
|---------|---------|---------|------|
| `intent:config` | Hash | 5分钟 | 意图配置缓存 |
| `intent:keywords` | Set | 5分钟 | 关键词集合 |

### 4.3 用户信息缓存

缓存用户基本信息。

| Key模式 | 数据类型 | 过期时间 | 说明 |
|---------|---------|---------|------|
| `user:info:{line_user_id}` | Hash | 1小时 | 用户信息缓存 |

### 4.4 任务锁

防止定时任务重复执行。

| Key模式 | 数据类型 | 过期时间 | 说明 |
|---------|---------|---------|------|
| `task:lock:{task_id}` | String | 1小时 | 任务执行锁 |

### 4.5 限流计数器

API调用频率限制。

| Key模式 | 数据类型 | 过期时间 | 说明 |
|---------|---------|---------|------|
| `rate:limit:{user_id}` | String | 1分钟 | 用户请求计数 |

-- 001_initial_schema.sql

-- 启用UUID扩展
CREATE EXTENSION IF NOT EXISTS "uuid-ossp";

-- 创建更新时间触发器函数
CREATE OR REPLACE FUNCTION update_updated_at_column()
RETURNS TRIGGER AS $$
BEGIN
    NEW.updated_at = NOW();
    RETURN NEW;
END;
$$ language 'plpgsql';

-- 创建用户表
CREATE TABLE users (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    line_user_id VARCHAR(64) NOT NULL,
    display_name VARCHAR(255),
    picture_url VARCHAR(512),

CREATE INDEX idx_users_status ON users(status);
CREATE TRIGGER update_users_updated_at BEFORE UPDATE ON users FOR EACH ROW EXECUTE FUNCTION update_updated_at_column();

CREATE TABLE conversations (
